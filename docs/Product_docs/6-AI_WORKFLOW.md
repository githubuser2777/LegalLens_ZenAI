# Hoạt động 6 - AI Workflow

# Mục tiêu

Xác định AI được sử dụng như thế nào trong toàn bộ vòng đời phát triển sản phẩm.

AI không thay thế nhóm phát triển.

AI đóng vai trò công cụ hỗ trợ.

---

# 1. Brainstorm Ý Tưởng

Prompt:

Suggest software product ideas that solve real-world problems using AI and RAG.

AI hỗ trợ:

* Sinh ý tưởng
* Gợi ý thị trường
* Phân tích user pain points

Kết quả:

LegalLens AI được lựa chọn.

---

# 2. Product Discovery

Prompt:

Who are the target users of a contract analysis platform?

AI hỗ trợ:

* Phân tích user
* Phân tích nhu cầu
* Viết user stories

---

# 3. Product Vision

Prompt:

Create a product vision using Moore Vision Template.

AI hỗ trợ:

* Vision Statement
* Product Positioning
* Value Proposition

---

# 4. MVP Planning

Prompt:

Prioritize features for a contract analysis MVP.

AI hỗ trợ:

* Must Have
* Should Have
* Could Have

---

# 5. UI/UX Design

Prompt:

Create dashboard ideas for a legal contract analysis platform.

AI hỗ trợ:

* Wireframe
* Layout
* User Flow

---

# 6. Software Architecture

Prompt:

Design a scalable architecture for a RAG-based contract analysis platform.

AI hỗ trợ:

* Frontend Structure
* Backend Structure
* RAG Pipeline

---

# 7. Coding

Prompt:

Implement PDF upload endpoint using FastAPI.

AI hỗ trợ:

* API Development
* Boilerplate Code
* Refactoring

# 8. Testing

Prompt:

Generate test cases for PDF upload and RAG retrieval.

AI hỗ trợ:

* Unit Tests
* Integration Tests

# 9. Documentation

Prompt:

Generate API documentation and README sections.

AI hỗ trợ:

* README

# 10. Evaluation

Prompt:

Identify weaknesses in our MVP.

AI hỗ trợ:

* Risk Assessment
* Feature Gaps
* Improvement Suggestions

# Rủi ro khi sử dụng AI
