# Step 03 - Personas and User Journey

## Purpose

Before defining features and requirements, the team must understand who the users are, what problems they face, and how they will interact with the product.

This document defines the primary personas and the main user journey for LegalLens AI.

---

# User Personas

### Profile

* Name: Minh Nguyen
* Age: 20
* Occupation: University Student

### Background

Minh is looking for accommodation near university and frequently encounters rental agreements that contain legal language he does not fully understand.

### Goals

* Understand rental agreements before signing
* Avoid losing deposits
* Identify important obligations
* Make informed decisions

* Contracts are long and time-consuming to review
* Important risks may be hidden in lengthy documents

# User Journey

## Primary Journey: Contract Risk Review

### Step 1: Receive Contract

The user receives a contract from a landlord, employer, client, or service provider.

### Step 2: Upload Contract

The user uploads the PDF document to LegalLens AI.

### Step 3: Contract Processing

The system extracts text and prepares the document for analysis.

### Step 4: Review Contract Overview

The user reviews basic contract information and document structure.

### Step 5: Analyze Risks

The user clicks the "Analyze Risks" button.

### Step 6: AI Risk Analysis

The system identifies potentially risky clauses and categorizes them.

Examples:

* Automatic renewal
* Deposit forfeiture
* Early termination penalties
* Data sharing permissions

### Step 7: Review Findings

The user reviews risk explanations and severity levels.

### Step 8: Verify Citations

The user opens the original clauses referenced by the analysis.

### Step 9: Make Decision

The user gains a better understanding of the contract and decides whether to proceed, negotiate, or seek additional advice.

---

# Key Insights

The primary value of LegalLens AI is not replacing legal professionals.

Instead, the platform helps users:

* Understand contracts faster
* Focus on important sections
* Detect potential risks
* Make more informed decisions

The AI component serves as a supporting tool rather than the core product itself.
